## Step 1 — FPU 모듈 동작 확인 (RTL Simulation)

### 1.1 목표
제공된 IEEE-754 단정밀도 FPU 모듈(adder/multiplier/divider)의 stb/ack handshake 사용법을 익히고,
사칙연산을 제어하는 `fpu_top.v`의 최소 동작을 RTL 시뮬레이션으로 검증한다.

### 1.2 작성한 코드 — `fpu_top.v`
- 입력: `clk, rstnn, var_x, var_y, request_fadd/fsub/fmult/fdiv` (8개), 출력: `var_z`, `done`.
- `fpu_adder` / `fpu_multiplier` / `fpu_divider`를 **각각 1개씩만** 인스턴스화(중복 금지).
- **뺄셈(fsub)은 별도 모듈 없이 구현**: `x - y = x + (-y)` 이므로 가산기에 넣기 전
  피연산자 b의 **부호비트(bit 31)를 반전**하여 adder로 처리한다.
- 각 모듈의 `input_a_stb/input_b_stb`(데이터 준비) ↔ `input_a_ack/input_b_ack`(수신 완료),
  `output_z_stb` ↔ `output_z_ack` handshake를 빠짐없이 사용한다.
- 연산 완료 시 `done`을 1-cycle 펄스로 출력하여 상위(IP_TOP)가 결과 시점을 알 수 있게 한다.

### 1.3 검증 결과 (로컬 iverilog)
입력 x=14.53(0x41687AE1), y=87.91(0x42AFD1EC) 기준, 4종 연산이 IEEE-754 기대값과 일치:

| 연산 | 결과(float) | hex |
|------|------------|-----|
| fadd (x+y)  | 102.44   | 0x42CCE148 |
| fsub (x−y)  | −73.38   | 0xC292C290 |
| fmult (x×y) | 1277.33  | 0x449FAAA2 |
| fdiv (x÷y)  | 0.165283 | 0x3E293FDC |

## Step 2 — FPU IP 설계 + fpu_test (RTL Sim → FPGA)

### 2.1 목표
`fpu_top`을 APB 슬레이브 IP(`IP_TOP.v`)에 통합하여 프로세서(ORCA)에 연결하고,
사칙연산 테스트 앱(`fpu_test.c`, 수정 금지)으로 동작을 검증한다.

### 2.2 작성한 부분 — `IP_TOP.v` (빈칸)
레지스터 맵: `0x0`=x, `0x4`=y, `0x8`=z(결과), `0xC` W=fadd/R=fsub, `0x10` W=fmult/R=fdiv.
채운 핵심 3가지:

1. **결과 반영**: APB write가 아닌 경우, FPU 완료 시 결과를 z에 적재
   ```verilog
   else if(fpu_done) var_z <= fpu_z;
   ```
2. **`fpu_top` 인스턴스 + 시작펄스/Busy FSM**: 레벨로 유지되는 request(APB wait 동안)를
   **1-cycle start 펄스**로 변환하고, 연산 중 `busy`를 유지. 같은 접근에서 재트리거되지 않도록
   `op_serviced` 플래그로 1회만 수행.
3. **`rpready_set` wait-state**: 연산 주소(0xC/0x10) 접근 중 결과가 나오기 전까지
   `rpready=0`으로 APB wait-state를 만들고, 완료 시 `1`로 복귀(APB `pready` 개념).
   ```verilog
   if(rpsel && op_addr && !op_serviced) rpready_set <= 0; // 연산중 -> wait
   else                                 rpready_set <= 1; // 정상/완료
   ```

### 2.3 SoC 연결 (글루코드)
RVX `make syn`이 생성한 `user/rtl/include/kmeans_fpu_user_region.vh`의 주석 템플릿을
`IP_TOP`(i_test1) 인스턴스로 연결하여 uNoC(APB)와 special IP를 결선했다.
메모리 맵상 `i_test1` 슬레이브 base = `0xE2020000`.

## Step 3 — K-means clustering에 FPU IP 적용 (RTL Sim → FPGA)

### 3.1 목표
K-means 응용의 소프트웨어 실수 연산을 설계한 FPU IP의 API로 대체하고,
(1) 군집 결과가 소프트웨어 버전과 동일한지, (2) 가속 효과(total tick)를 확인한다.

### 3.2 FPU API (fpu_test.c 패턴 재사용)
`k_means_oled.c` 상단에 헬퍼를 추가했다: `set_x/set_y/get_z`, `perform_fadd/fsub/fmult/fdiv`,
그리고 한 번에 호출하는 `fpu_add/fpu_sub/fpu_mult/fpu_div(a,b)`.
(write 0xC=fadd, read 0xC=fsub, write 0x10=fmult, read 0x10=fdiv; 결과는 read 0x8=get_z)

### 3.3 대체한 실수 연산 4곳 (사칙연산 4종 모두 사용)
1. 거리 계산 `dis += sqr((float)data[i][l] - means[j][l])`
   → `diff=fpu_sub(...)`, `sq=fpu_mult(diff,diff)`, `dis=fpu_add(dis,sq)`  (**fsub·fmult·fadd**)
2. 새 mean 누적 `temp[group[i]][j] += (float)data[i][j]`
   → `fpu_add(...)`  (**fadd**)
3. 평균 `temp[i][j] /= count[i]`
   → `fpu_div(temp[i][j], (float)count[i])`  (**fdiv**)
4. 수렴 판정 `ABS(temp[i][j] - means[i][j])`의 뺄셈
   → `ABS(fpu_sub(temp[i][j], means[i][j]))`  (**fsub**)

또한 `profiling_start/end("K-means clustering")` 구간 뒤에 `profiling_print()`를 추가해
clustering 루프의 total tick을 출력하도록 했다.

### 3.4 결과 (1) — 군집 결과가 소프트웨어와 동일 (RTL Sim, num_data=5)
| 항목 | 소프트웨어(float) | FPU IP 적용 |
|------|------------------|-------------|
| means[0] | (76.50, 16.00) | (76.50, 16.00) |
| means[1] | (48.00, 39.00) | (48.00, 39.00) |
