## Step 1. FPU 모듈 동작 확인 (RTL Simulation)

IEEE-754 단정밀도 FPU 모듈(adder/multiplier/divider)로 사칙연산을 제어하는 fpu_top.v를 작성하고, 테스트벤치로 fadd/fsub/fmult/fdiv 결과를 IEEE-754 기대값과 비교하여 RTL 시뮬레이션으로 검증하였다.

### 1.1 작성한 코드 (fpu_top.v 핵심)

덧셈/곱셈/나눗셈은 각 모듈을 하나씩만 인스턴스화하고, 뺄셈은 x − y = x + (−y) 이므로 피연산자 b의 부호비트를 반전하여 가산기로 처리한다. 아래는 연산 선택(FSM) 부분이다.

```verilog
case (state)
    S_IDLE: begin
        a_done <= 1'b0; b_done <= 1'b0;
        if (request_fadd) begin
            op_sel <= OP_ADD; oper_a <= var_x; oper_b <= var_y;                     state <= S_RUN;
        end
        else if (request_fsub) begin
            // x - y = x + (-y) : invert sign bit of y
            op_sel <= OP_ADD; oper_a <= var_x; oper_b <= {~var_y[31], var_y[30:0]}; state <= S_RUN;
        end
        else if (request_fmult) begin
            op_sel <= OP_MUL; oper_a <= var_x; oper_b <= var_y;                     state <= S_RUN;
        end
        else if (request_fdiv) begin
            op_sel <= OP_DIV; oper_a <= var_x; oper_b <= var_y;                     state <= S_RUN;
        end
    end
    S_RUN: begin
        if (sel_a_ack) a_done <= 1'b1;
        if (sel_b_ack) b_done <= 1'b1;
        if (sel_z_stb) begin var_z <= sel_z; done <= 1'b1; state <= S_IDLE; end
    end
    default: state <= S_IDLE;
endcase
```

세 모듈은 각각 하나씩만 인스턴스화한다(중복 없음).

```verilog
fpu_adder      u_adder ( .clk(clk), .rstnn(rstnn),
    .input_a(in_a), .input_a_stb(add_a_stb), .input_a_ack(add_a_ack),
    .input_b(in_b), .input_b_stb(add_b_stb), .input_b_ack(add_b_ack),
    .output_z(add_z), .output_z_stb(add_z_stb), .output_z_ack(add_z_ack) );
fpu_multiplier u_mult  ( ...mul_*... );
fpu_divider    u_div   ( ...div_*... );
```

## Step 2. FPU 모듈을 적용한 IP 설계 (FPGA Prototyping)

fpu_top을 APB 슬레이브 IP(IP_TOP.v)에 통합하여 ORCA 프로세서에 연결하고, 사칙연산 테스트 앱 fpu_test.c로 FPGA에서 동작을 검증하였다. i_test1 슬레이브 base 주소는 0xE2020000 이다.

### 2.1 IP_TOP.v 작성한 부분

레지스터 맵은 0x0=x, 0x4=y, 0x8=z(결과), 0xC(쓰기=fadd / 읽기=fsub), 0x10(쓰기=fmult / 읽기=fdiv) 이다. 작성한 핵심은 다음 네 가지이다.
1. FPU 연동 신호 선언(결과/완료/요청 펄스/busy 등).
2. FPU 연산 완료(fpu_done) 시 결과를 z 레지스터에 반영.
3. 레벨로 유지되는 요청을 1-cycle 시작 펄스로 변환하는 busy FSM과 fpu_top 인스턴스(op_serviced로 재트리거 방지).
4. 연산 주소 접근 중 결과가 나올 때까지 rpready=0으로 APB wait-state 생성.

```verilog
// ② var_z에 FPU 결과 반영
always@(posedge clk, negedge rstnn) begin
    if(rstnn==0)         var_z <= 0;
    else if(we_z==1'b1)  var_z <= rpwdata;
    else if(fpu_done)    var_z <= fpu_z;   // FPU 완료: 결과 적재
end

// ④ rpready wait-state
assign rpready = rpready_set;
always@(posedge clk, negedge rstnn) begin
    if(rstnn == 0)                            rpready_set <= 1;
    else if(rpsel && op_addr && !op_serviced) rpready_set <= 0; // 연산중 -> wait
    else                                      rpready_set <= 1; // 정상/완료
end
```

③ 시작펄스/busy FSM과 fpu_top 인스턴스, 글루코드(i_test1 = IP_TOP) 전체는 `보고서.docx` 및 `step2_code.zip`(user/rtl/src, kmeans_fpu_user_region.vh)에 수록.

## Step 3. K-means clustering에 IP 적용 (FPGA Prototyping)

K-means 응용의 소프트웨어 실수 연산을 설계한 FPU IP의 API로 대체하고, 군집 결과가 소프트웨어와 동일한지와 FPU 적용 전·후 가속효과를 확인하였다.

### 3.1 API로 대체한 부분

fpu_test.c의 set_x/set_y/get_z 패턴으로 fpu_add/fpu_sub/fpu_mult/fpu_div(a,b) 래퍼를 만들어 사용하였다.

```c
static inline float fpu_add (float a, float b){ set_x(a); set_y(b); perform_fadd();  return get_z(); }
static inline float fpu_sub (float a, float b){ set_x(a); set_y(b); perform_fsub();  return get_z(); }
static inline float fpu_mult(float a, float b){ set_x(a); set_y(b); perform_fmult(); return get_z(); }
static inline float fpu_div (float a, float b){ set_x(a); set_y(b); perform_fdiv();  return get_z(); }
```

k_means_oled.c에서 실수 연산 네 곳을 아래와 같이 대체하였다(사칙연산 4종 모두 사용).

```c
// (1) 거리계산 sqr((float)data - means) : fsub, fmult, fadd
float diff = fpu_sub((float)data[i][l], means[j][l]);
float sq   = fpu_mult(diff, diff);
dis        = fpu_add(dis, sq);
// (2) 새 mean 누적 : fadd
temp[group[i]][j] = fpu_add(temp[group[i]][j], (float)data[i][j]);
// (3) 평균 : fdiv
temp[i][j] = fpu_div(temp[i][j], (float)count[i]);
// (4) 수렴 판정 뺄셈 : fsub
if(ABS(fpu_sub(temp[i][j], means[i][j])) > 0.0001)
```
